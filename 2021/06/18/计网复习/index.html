发表于 2021-06-18 更新于 2021-07-01 分类于 课程 阅读次数： Valine：

概述

网络概述

网络：节点和链路（Node and Link）

互联网：多个网络互联起来

Internet：全球最大的，开放的采用TCP/IP协议的网络

ISP: Internet Service Provider（电信、移动、联通……）ISP继续分层。

RFC(request for comments)：一小部分为因特网标准。

三种交换

电路交换

建立链接

通话

释放链接

效率太低，不适用计算机交换

分组交换（报文交换升级）

分组之后添加首部

可以随时发送，不需要先建立链接

计算机网络定义

自治系统及其链接（见作业）

分类

技术，使用者，传输介质，拓扑结构，覆盖范围

性能指标

速率

带宽

从某一点到另一点，所能通过的最高数据率（出口带宽）单位也是b/s

吞吐量

单位时间通过某个网络的数据量

时延

发送时延

分组长度/发送速率

传播时延

信道长度/电磁波传播速率

处理时延

时延带宽积（以比特为单位的链路长度）

传播时延$\times$带宽

往返时间RTT

利用率

信道利用率：有百分之几的时间是被利用的（一般不超过50%）

网络利用率：全网络利用率的加权平均

丢包率

分组误码，网络拥塞

*计算机网络体系结构

OSI：物理，链路，网络，运输，会话，表示，应用；我们学习把会话，表示省掉

TCP/IP：接口层，网际层，运输层，应用层

说穿了，只有网际层（IP)，运输层(TCP,UDP)，应用层(HTTP,STMP;DNS,RTP)

分层的原因

把大问题划分为小问题

各层次作用

物理层

传输媒介，物理接口，信号表示

链路层

标识主机（MAC地址），区分出地址和数据，协调主机争用主机

网络层

IP，前三个数字标识网络

路由选择

运输层

进程之间的通信问题

处理传错的问题

应用层

特定的网络应用

我的理解：首先确定怎么交流，手语？口语？01？（物理层）；然后是一个村里面交流（链路层）；因为跨村交流（网络层）；万一信息再传的过程中搞错了怎么办？（运输层）；最后对应某一件具体的事采用啥规矩去办（应用层）

相关术语

实体：硬件或软件进程

对等实体：层次相同的实体

协议：控制两个对等实体的通信规则集合

协议三要素：

语法：信息格式

语义：双方要完成的操作

同步：时序关系

服务：本层可以对上一层提供服务

服务访问点：相邻两层实体交换信息的逻辑接口，用于区分服务类型。

数据链路层：类型

网络层：协议字段

运输层：端口号

协议数据单元PDU：对等层次之间传送的数据包

物理层：比特流

链路层：帧

网络层：分组

运输层：TCP报文段（segment）/UDP用户数据报（datagram）

应用层：报文（message）

服务数据单元SDU：同一系统中，层与层之间的数据包

物理层

奈氏：$2W \ baut = 2W \ 码元/s= 2Wlog_2(X)$

香农公式：$c=Wlog_2(1+\frac SN)$

链路层

链路：节点到节点的物理线路

数据链路：把实现的通信协议的硬件和软件加到链路上

链路层以帧传输

主要是三个问题：封装成帧，差错检测，可靠传输

以太网媒体接入控制采用：CSMA/CD

802.11局域网媒体接入控制协议：CSMA/CA

封装成帧

在上层交付的PDU添加帧头和帧尾，帧头和帧尾包含控制信息，帧定界就是它们的作用之一。

透明数据传输：对上层的数据没有限制。

高级数据链路控制协议：HDLC用帧头和帧尾的标志字段作为帧定界符，其值为01111110；HDLC为了实现透明传输，在连续5个1后面插入0。

MTU：帧的数据部分长度上限。

差错检测

奇偶校验

在数据后面添加以为奇偶校验位，1为奇

循环冗余检测CRC

约定好一个生成多项式

发送方基于待发送的数据和生成多项式计算出差错检测码（冗余码），将其添加到待传输数据的后面一起传输。

接收方通过生成多项式来计算数据是否产生误码。

可靠传输

比特差错（链路），分组丢失、失序、重复

停止-等待协议SW

DATA丢失，设置超时重传

ACK丢失导致重复，DATA添加序号

ACK迟到，重发DATA，再发一次ACK导致发送方不知道是哪个ACK，ACK添加序号

信道利用率：$U=\frac {T_D}{T_D+RTT+T_A} \approx \frac {T_D}{T_D+RTT}$

GBN

发送方：

发送窗口尺寸:$1 \lt W_T \le 2^n-1 $，n是构成分组序号的比特数。

等于1退化为SW，大于$2^n-1$接收方无法分辨新旧数据

接收方：

接收窗口为1

SR

接收方窗口大于1，小于等于发送窗口，如果大于，接收方无法分辨新旧数据。

接收方每次都会发一个确认帧

PPP

为点对点传输提供一个标准方法。

对各协议数据报封装成帧

链路控制协议LCP

一套网络控制协议NCPs

透明传输：异步传输采用字节填充法，同步传输采用比特填充法

媒体接入控制

静态划分

信道复用：

复用：通过一条物理线路，同时传输多路用户信号

当网络传输容量大于多条单一信道传输的总通信量时，可利用复用技术再一条物理线路上建立多条通信信道来充分利用传输媒体带宽。

频分FDM

所用用户占用不同的频带资源

时分TDM

所用用户在不同的时间占用同样的频带宽度

波分WDM

就是光的FDM

码分CDM(CDMA)

Code Division Multiple Access

每一个比特时间再划分为m个短的间隔，成为码片，通常m取64或128

用CDMA的每一个站指派一个唯一的m bit码片序列

发比特1，就发自己的那个码片序列，发比特0，就发自己码片序列的反码

码片序列挑选原则：

不同，相互正交

动态接入

CSMA/CD

Carrier Sense Multiple Access/Collision Detection

载波监听多址接入/碰撞检测

多址接入MA：多个站链接在一条总线上

载波监听CS：每个站发送帧之前要先检测一下总线上是否有其他站点在发送帧（先听后说）

如果检测到总线空闲96比特时间就发送这个帧

碰撞检测CD：边说边听

最小帧长为64字节

CSMA/CA

Carrier Sense Multiple Access/Collision Avoidance

802.11无线局域网使用CSMA/CA协议,再CSMA上增加一个碰撞避免CA

由于不可能避免避免所有碰撞，使用数据链路层确认机制（SW）保证数据被正确接收

媒体接入控制方式：

DCF：分布式协调功能Distributed Coordinateion Function。 在DCF下，没有中心控制站点，每个站点使用CSMA/CA争用信道来获取发送权。

PCF：点协调功能

IFS：帧间间隔InterFrame Space，所有的的站点必须在持续检测到信道空闲一段指定时间后才能发送帧

高优先级帧等待时间更短

SIFS：最短帧间间隔28$\mu s$

DIFS：DCF帧间间隔128$\mu s$，在DCF方式中发送数据帧和管理帧

允许信道预约：RTS请求发送，CTS允许发送

MAC,IP,ARP

MAC

Mediea Access Control

IP地址时TCP/IP体系结构的网际层所使用的地址，ARP协议属于TCP/IP体系结构的网际层，作用是将已知设备所分配的IP，获取设备的MAC

MAC地址是对各个接口的唯一标识

多播MAC地址，第二个16进制数是奇数。

IP

网络编号，主机编号

ARP

ARP高速缓存表，ARP请求报文（广播），响应报文为单播帧

ARP只能在一段链路上使用

集线器与交换机

集线器只工作在物理层，只是简单的转发，不进行碰撞检测，使用集线器的星型网络实际上可以看作是总线网。

交换机工作在全双工方式；有并行性，可以同时多对接口连通，没有碰撞；只工作在链路层，收到帧以后，在帧交换表查找，帧的目的MAC地址和对应的接口号。

STP

生成树协议STP，为了避免震荡、网络风暴等问题

VLAN

交换机接口分为Trunk和Access

网络层

实现网络互联。

主要问题：可靠传输、寻址、路由选择

提供两种服务

面向链接的虚电路服务

建立网络层链接：VC虚电路Virtual Circuit

无连接数据报服务

可靠通信由用户主机保证

不需要建立网络层链接

每个分组可以走不同的路径

每个分组首部必须携带主机的完整地址

IPv4

分类编址

A：0开头，8位

B：10开头，16位

C：110开头，24位

D：1110开头，多播地址；E：1111开头

ABC才可以分配给主机、路由器接口

最小的本地环回测试地址：127.0.0.1

划分子网

引入掩码

无分类编址

无分类路由选择CIDR，Classless Inter-Domain Routing

/20：前20个比特拿来当网络号

路由聚合（超网）：

找共同前缀

超时重传时间：

$RTO=RTT_S+4 \times RTT_D$

加权平均往返时间：

$RTT_{S1}=RTT_1$

$RTT_S=(1-\alpha)RTT_S+\alpha \times RTT,(\alpha=0.125)$

RTT偏差的加权平均RTTD：

$RTT_{D1}=RTT_1 \div 2$

$RTT_D=(1-\beta)RTT_D+\beta \times |RTT_S-RTT|,(\beta = 0.25)$

路由选择协议

RIP

距离直连为1，好路由是通过路由器最少的路。

只和相邻路由器交换信息

刚开始工作，只知道自己到直连网络的距离为1

每个路由器和相邻路由器交换信息

每个路由器都知道到本AS内各个网络的最短距离和下一跳地址，称为收敛

OSPF

每个路由器都会产生链路状态通告LSA，Link State Advertisement，LSA中包含以下内容：

直连网络链路状态信息

邻居路由器链路状态信息

LSA被封装在链路状态更新分组LSU中，采用洪泛法发送

使用OSPF的路由器都有一个链路状态数据库LSDB，用于存储LSA

比如有R1,R2两个路由器：

周期性发送问候分组

数据库描述分组

如果R1发现缺少某些链路状态，R1发送链路状态请求分组

R2收到请求，发链路状态更新分组给R1

R1再发链路状态确认分组

每30分钟，或者发生变化的时候，发送链路状态更新分组

OSPF在多点接入网络中的路由器邻居关系：

选举指定路由器DR，Designated router和BDR，Backup Designated router

所有的非DR/BDR之和DR/BDR建立邻居关系

OSPF将AS分为更小的范围

BGP

在配置BGP时，每个自治系统的管理员要选择至少一个路由器作为“BGP发言人”

BGP发言人要交换路由信息，必须建立TCP连接，端口号为179

在TCP交换BGP报文建立BGP会话

BGP发言人除了运行BGP外，还必须运行自己所在自治系统所使用的内部网关协议，比如RIP,OSPF

ICMP

为了更有效的转发IP数据报，使用网际控制报文协议

差错报告

终点不可达：路由器或主机不能交付数据报的时候，就向源点交付终点不可达报文。

源点抑制：路由器或者主机由于拥塞而丢弃数据，向源点发送抑制报文，使源点放慢发送速率

时间超过：TTL=0，当终点在预先规定的时间内不能收到一个数据报的全部数据片时，把已知收到的数据报片都丢弃，并发送时间超过报文

参数问题：检验和有问题，出现误码，丢弃，并发送参数问题

改变路由：发现还有更好的路

ICMP差错报告报文不再发送ICMP差错报告报文

对第一个分片的数据报片的所有后续数据报片都不发送ICMP差错报告报文

多播地址、特殊地址不发送

询问报文

回送请求和回答：

ICMP回送请求报文时由主机或路由器向一个特定的目的主机发出的询问

用来测试目的站是否可达

时间戳请求和：

进行始终同步和测量时间

分组网间探测PING

用来测试主机或路由器间的连通性

特定流的策略表项集合，负责数据包的查找转发

分为

TCP流量控制

就是让别人别发太快了，TCP接收方利用自己的接收窗口大小，来限制发送方发送窗口的大小，TCP发送方收到接收方的0窗口通知之后，启动持续计时器。持续计时器超时后，向接收方发送0窗口探测报文

TCP拥塞控制

拥塞窗口cwnd维护原则：只要网络没有出现拥塞，就一直增大，出现了拥塞就减小

判断拥塞的依据：没有按时收到确认报文

发送方将拥塞窗口作为发送窗口swnd=cwnd

慢开始门限ssthresh

cwnd<ssthresh，使用慢开始算法；

cwnd>ssthresh，使用拥塞避免算法；

cwnd=ssthresh，两种算法都可以

慢开始，拥塞避免，快重传，快恢复

TCP超时重传时间的选择

不可直接使用某次测量得到的RTT样本来计算超时重传时间RTO

利用每次测量得到的RTT样本，计算加权平均往返时间RTTs

新的$new\ RTTs=(1-\alpha)\times old\ RTTs+\alpha \times new\ RTT\ sample \ (\alpha=0.125)$

RTT偏差的加权平均RTTD：$RTT_{D1}=RTT_1 \div 2 $

新的RTTD：$RTT_{D} = (1-\beta) \times oldRTT_D+\beta\times|RTT_S-newRTT\ sample|\ (\beta=0.25)$

超时重传时间：$RTO=RTT_S+4 \times RTT_D$

每重传一次，RTO翻倍
